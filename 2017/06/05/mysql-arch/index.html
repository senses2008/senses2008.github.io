MySQL存储引擎比较

发表于 2017-06-05

MySQL存储引擎

Innodb

面向OLTP数据库应用

  • 支持事务
  • 行锁设计
  • 支持外键
  • 支持非锁定读（默认读操作不会产生锁）
  • 通过MVCC（多版本并发控制）获得高并发性能
  • 实现了SQL标准的4种隔离级别，默认为REPEATABLE级别
  • 使用next-key-locking的策略来避免幻读（phantom）。
  • 插入缓冲 insert buffer
  • 二次写 double write
  • 自适应哈希索引 adaptive hash index
  • 预读 read ahead
  • 聚集索引 clustered

MyISAM

面向OLAP数据库应用

  • 不支持事务
  • 表锁设计
  • 支持全文索引
  • 缓冲池只缓存索引文件，不缓存数据文件
  • MySQL5.0之前默认支持的表大小为4GB
  • 5.0之后默认支持256TB的单表数据

NDB

  • 集群存储引擎，类似于Oracle的RAC集群。
  • 数据全放在内存中

Memory

  • 之前叫HEAP存储引擎
  • 数据放在内存中
  • 适合存储数据的临时表
  • 使用哈希索引
  • 表锁设计
  • 并发性能差
  • 不支持TEXT和BLOB列类型

Archieve

  • 只支持INSERT和SELECT
  • 压缩比 1：10
  • 适合存储归档数据，如日志信息
  • 使用行锁

Maria

  • 新设计用来代替MyISAM
  • 支持缓存数据和索引文件
  • 行锁设计
  • 提供了MVCC功能
  • 支持事务和非事务安全选项
  • 更好的BLOB字符类型处理性能
