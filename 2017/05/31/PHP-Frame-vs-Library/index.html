PHP组件初识

发表于 2017-05-31

说实话，使用PHP开发近10年，一直都封闭在框架的思维下。
框架决定了项目后期维护和扩展的便利性，所以对框架的选型，往往是一个PHP项目很重要的一个抉择。
不过大多数项目或者开发者，其实并不会多么严苛的去对比，而更多是凭经验，喜好，流行而已。
但当项目后期遇到框架的局限时，就要背着历史包袱，自己去解决各种难题，这时候才会悔不当初，为啥没有选更NB的框架。。。

什么是组件？

“组件”其实就是打包的代码，用于帮你解决应用中的某个具体问题，打包的代码可以是一些列相关的Class, Interface, Trait。这些代码放在一个共同的namespace下。

为什么要使用组件？

对大多数PHP的开发者来说，组件还是一个比较陌生的概念。因为大多数时候一个PHP项目，还是会去选择一个框架，或者自己来实现一个框架。我们会花很多时间研究一个框架，学习使用，了解框架的生态，每切换一个项目或者团队，都会有新的框架需要学习。不过当遇到框架解决不了的问题，就需要自己来开发，很难去集成自定义或者三方库。
